# 要件定義書

## 1.1 背景

業務委託エージェントとの案件やり取りは、現在Gmailで行っている。
1案件につき1メールスレッドで管理する運用にしているが、返信が数日後になることが多く、以下の課題がある。

* この案件がどのような内容だったか忘れる
* 今のステータスがわからなくなる
* 自分が返信する番なのか、相手の返信待ちなのかわからなくなる
* 面談予定・面談準備・結果待ちなどがGmailだけでは管理しづらい
* 面談時の想定質問、逆質問、メモ、対策を案件ごとに残しにくい

## 1.2 目的

Gmail上の業務委託案件を、案件単位で整理・管理できるアプリを作る。
主な目的は以下。

* 案件ごとのステータスを見える化する
* Gmailスレッドへすぐ戻れるようにする
* 次にやるべきアクションを明確にする
* 面談準備をAIで支援する
* 面談メモ・結果メモを案件ごとに残す
* 将来的には他ユーザーにも公開できる構成にする

## 1.3 対象ユーザー

### Phase 1

* 自分自身
* 業務委託案件をGmailで管理している個人

### Phase 2以降

* フリーランスエンジニア
* 業務委託案件を複数並行で進める人
* エージェントとのメール管理に課題がある人

## 1.4 開発方針

| 項目            | 方針                      |
| ------------- | ----------------------- |
| 初期方針          | 自分でドッグフーディング            |
| 将来方針          | 一般公開・課金も視野に入れる          |
| 認証            | Clerk                   |
| ログイン          | Googleアカウント             |
| 課金            | Stripe                  |
| Gmail連携       | 初期は手動貼り付け + Gmail URL保存 |
| Googleカレンダー連携 | 面談予定登録・リマインド連携          |
| AI            | 案件整理、次アクション提案、面談準備支援    |

## 1.5 MVP範囲

MVPでは以下を作る。

* 案件登録
* Gmail本文・必要部分の貼り付け
* Gmail URL保存
* AIによる案件情報整理
* 案件カンバン
* ステータス管理
* 次アクション管理
* 案件詳細
* 面談準備
* 面談メモ
* リマインド
* Googleカレンダー連携の基本機能
* Clerk認証

## 1.6 ステータス

案件ステータスは以下。

| ステータス  | 意味              |
| ------ | --------------- |
| 返信必要   | 自分が返信する必要がある    |
| 相手返信待ち | エージェント・企業側の返事待ち |
| 書類選考中  | 書類・経歴書などの選考中    |
| 面談調整中  | 面談日程を調整中        |
| 面談予定   | 面談日時が決まっている     |
| 結果待ち   | 面談後の結果待ち        |
| 保留     | 一旦保留            |
| 終了     | 辞退・見送り・条件不一致など  |

終了理由は別項目で持つ。

* 辞退
* 見送り
* 条件不一致
* 音信不通
* その他

## 1.7 機能要件

### 案件管理

* 案件を登録できる
* 案件名を保存できる
* エージェント名を保存できる
* 担当者名を保存できる
* 単価を保存できる
* 稼働率を保存できる
* 勤務地・リモート条件を保存できる
* 技術スタックを保存できる
* 案件概要を保存できる
* Gmail URLを保存できる

### AI整理

* 貼り付けたメール内容から案件情報を抽出する
* 案件概要を要約する
* 単価・勤務地・技術要素を抽出する
* 現在ステータス候補を提示する
* 次アクション候補を提示する
* ユーザーが内容を編集して保存できる

### カンバン

* ステータスごとに案件カードを表示する
* ドラッグ&ドロップでステータス変更できる
* プルダウンでもステータス変更できる
* 案件カードから詳細画面へ遷移できる
* 案件カードからGmailを開ける

### 面談準備

* 面談日時を保存できる
* 面談URLを保存できる
* 想定質問をAI生成できる
* 想定質問への回答案をAI生成できる
* 自分から聞きたいことをAI生成できる
* 懸念点・確認事項をAI抽出できる
* 面談戦略をAI生成できる
* 面談前チェックリストを表示できる
* すべて手編集できる

### 面談メモ

* 面談中メモを保存できる
* 面談後メモを保存できる
* 面談結果の印象を保存できる
* 次アクションを保存できる

### リマインド

* 返信必要のまま一定時間経過した案件を表示する
* 相手返信待ちが一定日数続いた案件を表示する
* 面談予定の前日にリマインドする
* 結果待ちが一定日数続いた案件を表示する

### Googleカレンダー連携

* 面談予定をGoogleカレンダーに登録できる
* 面談日時・面談URL・案件URL・Gmail URLをカレンダー予定に含める
* カレンダーイベントURLを案件に保存する

## 1.8 非機能要件

| 項目     | 要件                            |
| ------ | ----------------------------- |
| セキュリティ | ユーザーごとにデータを分離する               |
| 認証     | ClerkでGoogleログイン              |
| 可用性    | 個人利用MVPでは高可用性は必須ではない          |
| 拡張性    | 将来公開を見据え、ユーザーID単位で設計          |
| 保守性    | AI処理、案件管理、認証、課金を分離            |
| プライバシー | Gmail本文はユーザーが貼り付けたもののみ保存      |
| 外部連携   | Google Calendar、Stripeを段階的に導入 |
