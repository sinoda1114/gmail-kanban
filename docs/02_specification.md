# 仕様書

## 2.1 案件登録仕様

### 入力項目

| 項目        | 必須 | 内容           |
| --------- | -: | ------------ |
| Gmail URL | 任意 | 元メールスレッドのURL |
| メール本文     | 必須 | 最新メールまたは必要部分 |
| メモ        | 任意 | 登録時の補足       |

### 処理

1. ユーザーがメール内容を貼り付ける
2. 必要に応じてGmail URLを貼る
3. 「AIで整理する」を押す
4. AIが案件情報を抽出する
5. 確認画面でユーザーが編集する
6. 保存する

## 2.2 AI整理仕様

AIが抽出する項目は以下。

| 項目       | 内容                  |
| -------- | ------------------- |
| 案件名      | メール内容から推定           |
| エージェント名  | 会社名・サービス名           |
| 担当者名     | メール署名などから推定         |
| 案件概要     | 3〜5行程度              |
| 単価       | 月額・時給など             |
| 稼働率      | 週5、週3、50%など         |
| 勤務地      | フルリモート、出社、ハイブリッド    |
| 技術スタック   | React、Next.js、AWSなど |
| 契約期間     | 期間があれば抽出            |
| 開始時期     | 即日、7月開始など           |
| ステータス候補  | AI推定                |
| 次アクション候補 | AI提案                |

### ステータスは自動確定しない

AIは候補のみ提示する。

```text
AI推定：相手返信待ち
理由：こちらから返信済みで、相手の回答を待っている内容のため
```

ユーザーが確認して保存する。

## 2.3 カンバン仕様

### カラム

* 返信必要
* 相手返信待ち
* 書類選考中
* 面談調整中
* 面談予定
* 結果待ち
* 保留
* 終了

### 案件カード表示項目

| 項目       | 表示    |
| -------- | ----- |
| 案件名      | 表示    |
| エージェント名  | 表示    |
| ステータス    | 表示    |
| 次アクション   | 表示    |
| 最終更新日    | 表示    |
| Gmailを開く | ボタン表示 |

面談予定の場合は追加で表示。

* 面談日時
* 面談準備ボタン

### ステータス変更

以下の両方に対応する。

* ドラッグ&ドロップ
* プルダウン変更

変更時にはステータス履歴を保存する。

## 2.4 案件詳細仕様

案件詳細にはタブを持つ。

* 基本情報
* 面談準備
* 面談メモ
* 履歴

### 基本情報タブ

表示・編集項目。

* 案件名
* ステータス
* エージェント名
* 担当者名
* 単価
* 稼働率
* 勤務地
* 技術スタック
* 案件概要
* Gmail URL
* 次アクション
* リマインド日

## 2.5 面談準備仕様

### 面談情報

| 項目            | 内容                   |
| ------------- | -------------------- |
| 面談日時          | 日時                   |
| 面談URL         | Google Meet / Zoomなど |
| 面談形式          | オンライン / 対面           |
| 面談相手          | エージェント / 企業 / 技術担当   |
| Googleカレンダー連携 | イベントURL保存            |

### AI生成項目

* 想定質問
* 回答案
* 逆質問
* 懸念点
* 面談戦略
* 面談前チェックリスト

### 想定質問

質問ごとに以下を持つ。

* 質問文
* カテゴリ
* 重要度
* AI回答案
* 自分用回答
* メモ

### 逆質問

自分から聞く質問として以下のカテゴリで生成する（カッコ内はDB保存値）。

* 役割（role）
* 体制（team）
* 技術（tech）
* 働き方（work_style）
* 契約（contract）
* 選考フロー（selection_flow）

### 面談戦略

AIは以下を生成する。

* 強調すべき経験
* 注意すべき点
* 深掘りされそうな点
* 回答方針
* 確認すべきリスク

## 2.6 面談メモ仕様

保存項目（カッコ内は `interview_notes` のカラム）。

* 面談中メモ（during_note）
* 面談後メモ（after_note）
* 先方の印象（impression）
* 自分の温度感（own_temperature）
* 懸念点（concern）
* 次アクション（next_action）
* 結果ステータス（result_status・案件statusと同じ値集合）

## 2.7 リマインド仕様

| 条件          | 表示        |
| ----------- | --------- |
| 返信必要のまま1日経過 | 要対応       |
| 相手返信待ちが3日経過 | 催促候補      |
| 面談予定の前日     | 面談準備リマインド |
| 結果待ちが5日経過   | 進捗確認候補    |
| 保留が7日経過     | 再確認候補     |

初期はアプリ内表示のみ。
メール通知・プッシュ通知は後続対応。

## 2.8 Googleカレンダー仕様

### できること

* 面談予定をカレンダー登録
* 面談URLを登録
* 案件詳細URLを予定説明に追加
* Gmail URLを予定説明に追加
* カレンダーイベントURLを案件に保存

### MVPではやらないこと

* カレンダー予定の双方向同期
* 空き時間検索
* 自動日程調整
* Gmail本文から自動でカレンダー登録
